# Backlog del proyecto

Este backlog recoge las tareas y funcionalidades previstas para la plataforma de gestión de citas, agrupadas por áreas. Es un documento vivo que se irá actualizando a medida que avance el desarrollo.

## Funcionalidades principales (pendientes)
- Configuración de servicios por profesional (duración, precio, modalidad).
- Gestión de disponibilidad (franjas horarias, descansos, días bloqueados).
- Generación de huecos disponibles según reglas de agenda.
- Reserva de citas por parte del cliente.
- Integración con pasarela de pago (PayPal y Stripe en modo test).
- Confirmación de cita tras el pago.
- Envío de recordatorios automáticos.
- Panel del profesional (calendario, citas, clientes).
- Página pública del profesional (lista de servicios + disponibilidad).
- Sistema de autenticación para profesionales.
- Edición del perfil del profesional (logo, datos fiscales, textos legales).

## Tareas técnicas (pendientes)
- Definir arquitectura final (Node/Express o FastAPI).
- Definir modelo de datos y esquema de base de datos.
- Crear endpoints iniciales del backend.
- Estructura base del frontend.
- Configuración del entorno de desarrollo.
- Documentación técnica inicial.

## Ideas para futuras versiones (no incluidas en el MVP)
- Asignación automática de huecos inteligentes (tipo secretaria virtual).
- Módulo de tareas o seguimiento para clientes.
- Estadísticas avanzadas para el profesional.
- Integración con otras pasarelas de pago según región.
- App móvil simplificada.
